# 記号ハイライト ユーザースクリプト 設計書

## 概要

競技プログラミングのコード・問題文を読む際に、記号文字が視認しにくい問題を解決するためのTampermonkeyユーザースクリプト。ページ上の記号文字に色を付けて見やすくする。

## 技術スタック

- Tampermonkey ユーザースクリプト（JSファイル1つで完結）
- 純粋なDOM操作のみ（Tampermonkey固有APIは使わない。将来Chrome拡張へ移行しやすくするため）

## 対象サイト

以下のサイトで動作させる（`@match`に指定）:

- AtCoder: `https://atcoder.jp/*`
- Codeforces: `https://codeforces.com/*`

※ 必要に応じて追加可能

## ハイライト対象の記号

以下のカテゴリに分けて、カテゴリごとに異なる色を付ける:

| カテゴリ | 記号 | 色の方針 |
|---------|------|---------|
| 括弧 | `()` `{}` `[]` `<>` | 対応が分かるよう目立つ色 |
| 演算子 | `+` `-` `*` `/` `%` `=` `!` `&` `\|` `^` `~` | 括弧とは別の色 |
| 区切り | `;` `:` `,` `.` | 控えめだが判別できる色 |

具体的な色はスクリプト冒頭の定数で定義し、変更しやすくする。

## 処理フロー

1. ページ読み込み完了後に実行
2. ページ内の `<pre>` `<code>` タグ（コード表示領域）を対象にする
3. TreeWalker でテキストノードを走査
4. 正規表現で記号にマッチさせ、マッチ部分を `<span class="sh-bracket">` 等で囲む
5. span に対してインラインスタイルまたは動的に挿入した CSS で色を当てる

## 設計上の注意点

- テキストノードのみを対象にし、既存のDOM構造やイベントリスナーを壊さないこと
- シンタックスハイライト済みの要素（既に `<span>` で囲まれている中のテキストノード）にも対応すること
- 動的にコンテンツが追加されるケースに備え、MutationObserver での監視も入れる
- パフォーマンス: 大量のテキストノードがある場合でも重くならないよう、対象を `<pre>` `<code>` 内に限定する

## ファイル構成

```
symbol-highlighter.user.js  （単一ファイル）
```

ファイル内の構成:

```
==UserScript== メタブロック
├── @name, @match, @version 等
定数定義
├── 色設定（カテゴリごと）
├── 対象記号の正規表現
メイン処理
├── highlightSymbols(): テキストノード走査 & span置換
├── injectStyles(): CSS動的挿入
├── setupObserver(): MutationObserver設定
エントリポイント
├── injectStyles() → highlightSymbols() → setupObserver()
```
